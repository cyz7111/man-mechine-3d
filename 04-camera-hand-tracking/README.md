# 04 Camera Hand Tracking

这是系列里的第四个实验：用摄像头输入控制 3D 物体。

当前版本不依赖外部库，先用画面运动中心作为“手部位置”的简化代理。它不是精确的手部关键点识别，但非常适合先理解摄像头交互里的几个核心问题：权限、噪声、置信度、平滑、fallback。

## 你要观察什么

- 输入：摄像头画面里的移动区域，或指针模拟。
- 解析：系统根据前后帧差异计算运动中心。
- 输出：3D 物体的位置、旋转和发光强度变化。
- 反馈：状态、X/Y、置信度和模式同步更新。
- 容错：没有摄像头权限时，仍然可以用指针模拟。

## 运行方式

摄像头需要通过本地服务器访问：

```txt
http://localhost:4173/04-camera-hand-tracking/
```

如果浏览器阻止摄像头，可以打开“指针模拟”继续练习交互映射。

## 小任务

1. 把运动灵敏度调高，让轻微移动也能被识别。
2. 把运动灵敏度调低，减少误触。
3. 把 `setTracking()` 里的平滑系数从 `0.24` 改成 `0.08`，观察延迟。
4. 把物体的左右移动反向。
5. 当置信度低于 `20%` 时，让物体回到中心。

## 复盘问题

- 摄像头交互最不稳定的地方是什么？
- 置信度显示有没有帮助你理解系统状态？
- 平滑让操作更自然，还是让操作更迟钝？
- 用户没有摄像头权限时，fallback 应该怎么设计？

## 下一步升级

完成这个实验后，可以进入更真实的手部识别：

- 引入 MediaPipe Hands。
- 读取手腕、食指指尖、拇指指尖等关键点。
- 用张开/捏合判断不同模式。
- 加入误触保护，例如连续命中 3 帧才执行。
