# 长期路线图

## 阶段 1：建立交互直觉

重点不是复杂技术，而是让你开始观察：

- 用户给了什么输入？
- 系统如何理解输入？
- 系统给了什么反馈？
- 用户下一步是否知道该怎么做？

建议完成：

- `01-pointer-3d-card`
- `02-drag-3d-object`

## 阶段 2：从二维界面进入空间界面

开始使用真正的 3D 工具，并理解屏幕坐标、世界坐标、相机和物体之间的关系。

建议技术：

- Three.js
- React Three Fiber
- GSAP 或 spring 动画

建议完成：

- 可拖拽 3D 物体
- 可缩放、旋转、复位的模型查看器
- 带操作历史的 3D 编辑面板

## 阶段 3：多模态输入

让用户不只通过鼠标操作，还能通过语音、摄像头、手机传感器来控制系统。

建议技术：

- Web Speech API
- MediaPipe
- DeviceOrientation API
- WebRTC 基础

建议完成：

- 语音控制场景颜色和模式
- 手势控制物体旋转和缩放
- 手机作为遥控器控制桌面网页

## 阶段 4：人机协作

把 AI 加进来，但重点不是炫技，而是让 AI 的行为可理解、可修改、可撤销。

建议完成：

- AI 根据自然语言生成 3D 场景操作计划
- 用户确认、修改或撤销 AI 的操作
- AI 行为日志和可解释面板

## 每个作品的复盘模板

```md
# 实验复盘

## 输入

用户通过什么方式表达意图？

## 输出

系统发生了什么可见变化？

## 反馈

用户如何知道系统理解了自己？

## 问题

哪些地方不自然、不稳定或容易误解？

## 下一版

下一步只改一个最重要的问题。
```
