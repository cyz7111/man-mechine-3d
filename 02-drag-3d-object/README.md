# 02 Drag 3D Object

这是系列里的第二个实验：把“指针移动产生反馈”升级成“用户直接拖拽物体”。

## 你要观察什么

- 输入：用户按住物体并拖动。
- 映射：指针移动距离变成物体在舞台里的位置变化。
- 反馈：物体进入拖拽状态、角度轻微变化、数值面板更新。
- 目标：物体中心进入目标区后，目标和物体都会高亮。
- 恢复：点击复位按钮后，位置、次数和耗时归零。

## 运行方式

直接用浏览器打开：

```txt
index.html
```

## 小任务

1. 让物体只能水平移动。
2. 让物体只能垂直移动。
3. 把目标区换到左上角。
4. 把“命中目标”的判定从中心点改成整个物体进入目标区。
5. 记录哪一种判定更符合你的直觉。

## 复盘问题

- 按住物体时，你是否能明显感觉系统进入了拖拽状态？
- 拖动边界是否自然？
- 命中目标的反馈是否足够及时？
- 位置、距离、次数和耗时里，哪个指标最有帮助？
- 如果这是一个真实工具，你会增加撤销、吸附还是路径记录？
