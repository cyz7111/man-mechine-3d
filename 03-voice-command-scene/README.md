# 03 Voice Command Scene

这是系列里的第三个实验：用命令改变场景状态。

这里的重点是多模态交互。用户可以通过语音、文本输入或按钮发出同一种命令，系统用统一的命令解析逻辑改变场景。

## 你要观察什么

- 输入：语音、文本命令、按钮点击。
- 解析：系统把自然语言或快捷按钮转成明确动作。
- 输出：颜色、缩放、旋转状态发生变化。
- 反馈：状态栏、最后一次命令、指标面板同步更新。
- 容错：语音不可用时，文本和按钮仍然可用。

## 可用命令

中文：

- `变蓝`
- `变绿`
- `变红`
- `旋转`
- `暂停`
- `放大`
- `缩小`
- `复位`

英文：

- `blue`
- `green`
- `red`
- `rotate`
- `pause`
- `bigger`
- `smaller`
- `reset`

## 运行方式

如果直接打开 `index.html` 时语音不可用，可以用本地服务器打开：

```txt
http://localhost:4173/03-voice-command-scene/
```

语音识别能力取决于浏览器支持和权限设置。按钮和文本命令不受影响。

## 小任务

1. 增加一个新命令：`变黄`。
2. 增加一个新命令：`隐藏`，让物体透明度变低。
3. 增加一个新命令：`显示`，让物体恢复可见。
4. 把 `放大` 的步长从 `0.15` 改成 `0.25`，观察操作是否变得粗糙。物体变得更大了
5. 记录语音、文本、按钮三种输入里，哪一种反馈最清楚。

## 复盘问题

- 用户说错或系统没听懂时，反馈是否足够明确？
- 语音命令是否应该自动连续监听？
- 命令执行后是否需要可撤销？
- 多种输入方式共存时，哪个入口应该是主入口？
